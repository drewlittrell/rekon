# Governance

Rekon starts as a maintainer-led open-source project.

## Maintainer Responsibilities

- Preserve the public extension API as a documented contract.
- Keep built-in capabilities on the same SDK path as community capabilities.
- Review schema, artifact, and permission changes as public API changes.
- Keep generated publications separate from canonical truth artifacts.
- Document breaking changes in `CHANGELOG.md`.

## Decision Records

Architecture decisions live in `docs/adr/`.
