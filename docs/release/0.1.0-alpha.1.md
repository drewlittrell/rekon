# Rekon 0.1.0-alpha.1

This is the first publishable alpha of Rekon. Until packages are actually
published to npm, the recommended way to use Rekon is to build from source
(see "Install" below).

## What Rekon Is

Rekon is an open-source intelligence substrate for codebases: evidence in,
typed artifacts out, extensible capabilities around a shared repository
intelligence snapshot.

The substrate is local-first. `rekon` writes typed artifacts under
`.rekon/` and indexes them in an `IntelligenceSnapshot`. Built-in and
community capabilities project, evaluate, resolve, publish, learn, and act
around that shared snapshot using the same public SDK.

See [docs/strategy/north-star.md](../strategy/north-star.md) for the
durable plan.

## What's Included In `0.1.0-alpha.1`

- 7 kernel packages: `@rekon/kernel-artifacts`,
  `@rekon/kernel-evidence`, `@rekon/kernel-snapshot`,
  `@rekon/kernel-graph`, `@rekon/kernel-repo-model`,
  `@rekon/kernel-rulebook`, `@rekon/kernel-findings`.
- Public SDK: `@rekon/sdk`.
- Local runtime: `@rekon/runtime`.
- CLI: `@rekon/cli` (binary name `rekon`).
- 9 built-in capabilities: `@rekon/capability-js-ts`,
  `@rekon/capability-model`, `@rekon/capability-graph`,
  `@rekon/capability-policy`, `@rekon/capability-resolver`,
  `@rekon/capability-docs`, `@rekon/capability-memory`,
  `@rekon/capability-intent`, `@rekon/capability-reconcile`.
- Strategy documentation set (NorthStar, capability model, roadmap,
  classic-repo migration).
- Alpha release readiness scripts (`audit-package-exports`,
  `publish-dry-run`, `audit-license`, `install-smoke`,
  `install-tarball-smoke`).

All 19 packages share the same version (`0.1.0-alpha.1`) and the same
stability label (`experimental, public`). Symbols not re-exported from a
package root are `internal`.

## Install And Use From Source

Until packages are published to npm, use Rekon from a source checkout:

```sh
git clone https://github.com/drewlittrell/rekon.git
cd rekon
npm install
npm run build

node packages/cli/dist/index.js --help
```

If the packages have been published to npm (see "Publish Status" below),
the canonical install is:

```sh
npm install --no-save @rekon/cli
node node_modules/.bin/rekon --help
```

## Key CLI Flows

The alpha CLI implements the full local lifecycle:

```sh
node packages/cli/dist/index.js init --root <repo>
node packages/cli/dist/index.js observe --root <repo> --json
node packages/cli/dist/index.js project --root <repo> --json
node packages/cli/dist/index.js snapshot --root <repo> --json
node packages/cli/dist/index.js evaluate --root <repo> --json
node packages/cli/dist/index.js resolve preflight --root <repo> --path <file> --goal "<intent>" --json
node packages/cli/dist/index.js publish agents --root <repo>
node packages/cli/dist/index.js memory add --root <repo> --instruction "..." --path <path>
node packages/cli/dist/index.js memory list --root <repo> --json
node packages/cli/dist/index.js memory select --root <repo> --path <file> --goal "<intent>" --json
node packages/cli/dist/index.js intent work-order --root <repo> --path <file> --goal "<intent>" --json
node packages/cli/dist/index.js reconcile --root <repo> --operation docs_regeneration
node packages/cli/dist/index.js artifacts list --root <repo> --json
node packages/cli/dist/index.js artifacts show <id-or-type:id> --root <repo> --json
node packages/cli/dist/index.js artifacts validate --root <repo> --json
```

The 10-minute walkthrough is in
[docs/getting-started/first-10-minutes.md](../getting-started/first-10-minutes.md).

## Capability Authoring

Community capabilities use the same public SDK as built-ins:

- `defineCapability(...)` from `@rekon/sdk`.
- Declare roles, consumes, produces, permissions, invalidation, and
  Rekon compatibility.
- Validate with `validateCapability()` or
  `assertCapabilityConforms()` in tests.
- Register external capabilities via `.rekon/config.json`.

See [docs/extensions/authoring-capabilities.md](../extensions/authoring-capabilities.md)
and [examples/custom-capability](../../examples/custom-capability).

## Known Limitations

- No watcher or freshness engine beyond current artifact metadata.
- No package marketplace or discovery layer.
- No hosted dashboard or GitHub app.
- No source-writing reconciliation by default. Reconciliation produces
  artifacts and stays permission-gated.
- No full `codebase-intel-classic` behavioral parity. The classic repo
  is reference, dogfood target, and migration source only.
- Lightweight, dependency-free schema validation.

## Dogfood

A larger regression harness runs against any large JS/TS codebase
(intended use: `codebase-intel-classic`). It is opt-in and skips cleanly
when no path is configured:

```sh
REKON_DOGFOOD_CLASSIC_ROOT=/path/to/codebase-intel npm run test
```

## Architecture Rule

Lower layers may feed upper layers. Upper layers may not silently become
lower-layer truth. Docs are publications, not canonical truth. Memory
enriches resolver output; it does not rewrite ownership, rules, or
findings. Reconciliation remains permissioned and artifact-first; source
writes require explicit permission and explicit operator approval.
