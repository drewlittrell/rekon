# Capability Ontology

The capability ontology normalizes action and capability language found in a
repository. Rekon uses it to keep capability names, extracted phrases, and
policy hints consistent without turning heuristic language into canonical
architecture.

Ontology outputs are supporting artifacts. They can inform summaries,
suggestions, and resolver context; they do not rewrite evidence, ownership, or
rules by themselves.
