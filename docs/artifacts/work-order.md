# WorkOrder

`WorkOrder` describes scoped work to be performed. It is an artifact, not source
execution.

## Produced By

- intent and remediation actuators

## Common Fields

- `header`
- `goal`
- `scope`
- `tasks`
- `requiredChecks`
- `successCriteria`
- `risks`
- `inputRefs`

Work orders should make scope and verification obligations explicit.
